就绪通信模式

 　　就绪通信模式对发送操作调用时机有严格的要求：发送操作必须在对应的接收操作已经被调用（接收处理器已经做好准备，可以接收数据--所以称为就绪通信模式）的情况下，才能安全的启动发送操作，否则会给出错误消息。（这种方式可以用作对系统的监控和调试）。

　　MPI_RSEND( buf, count, datatype, dest, tag, comm)
　　IN buf
　　IN count
　　IN datatype
　　IN dest
　　IN tag
　　IN comm

　　int MPI_Rsend( void *buf, int count, MPI_Datatype datatype, int dest,
　　int tag, MPI_Comm comm)；

　　MPI_RSEND的参数含义和MPI_SEND完全相同。对就绪通信模式，也没有提供专门的接收函数，接收进程应该使用标准的MPI_RECV调用。
