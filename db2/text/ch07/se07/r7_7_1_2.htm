4.警报管理
��警报是预先定义好的，当某些SQL Server事件发生的时候服务器应该作出的反应。一般来说，服务器管理人员无法控制事件的发生与否，但可以控制当事件发生的时候应该执行的操作。
��在SQL Server Agent 中可以配置警报，使得当SQL Server 产生一个特定错误信息，或当系统的某个阈值达到指定条件时，将触发此警报。
例如，如果SQL Server运行时磁盘空间不够，就会有一个事件被写入Windows NT 事件日志中。可以设置让此事件触发一个警报。通过此警报，可以通知某个操作员磁盘空间不够。
��用系统性能阈值触发警报的例子。例如，可以在SQL Server用户连接数超过一个特定值时，产生一个警报。


��在Enterprise Manager中创建警报的步骤如下：
��(1) 展开指定的服务器节点；
��(2) 展开Management 节点；
��(3) 展开SQL Server Agent 节点；
��(4) 选中Alerts 节点，右击鼠标，从弹出的菜单中选择New Alerts 命令。也可以通过运行Create Alert Wizard来创建警报。
��可以在创建警报时，指定警报的名字，指定当错误发生时，用于触发警报所必需的错误消息或文字，指定在错误消息中出现什么样的文字，指定与某一事件相关的警报产生时，如何响应它等。如图片7-17。
