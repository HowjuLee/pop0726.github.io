1．使用SQL Server代理服务：
��SQL Server代理服务的使用包括：
��(1) SQL Server代理服务的配置；
��(2) 操作员管理；
��(3) 作业管理；
��(4) 警报管理。

2．操作员(Operator)管理
��操作员是负责维护一台或多台SQL Server服务器的权限实体。操作员可以是一个用户，也可以是多个用户。当SQL Server预先定义的事件发生时，SQL Server服务器可以通过电子邮件、寻呼机或Windows网络消息通知操作员。可以配置某个操作员，使之成为响应某一警报消息的接收者。
在Enterprise Manager中创建操作员的步骤如下：
��(1) 展开指定的服务器节点；
��(2) 展开Management节点；
��(3) 展开SQL Server Agent节点；
��(4) 选中Operators节点，右击鼠标，从弹出菜单中选择New Operators命令。

3．作业（Job）管理
��用户可以使用作业来定义将要执行一次或多次的管理任务，并监督该任务的完成情况，究竟是成功还是失败。作业可以在本地服务器和多台远程服务器上运行，可以按一定时间表运行，也可以通过警报来触发。
��SQL Server Agent是一个作业调度服务。作业可以包含一个或多个步骤。每个作业步骤可以是一条操作系统命令，或是一组Transact－SQL指令，也可以是ActiveX脚本语句等。
