　　70年代后期，随着CAD技术的发展，在不同系统之间交换设计意图(Deisn intent)的问题得到广泛重视，人们希望各CAD系统的数据直接通过计算机进行交换。
　　70年代初，CAM-I公司(Computer-Aided Manufacturing International, Inc.)委托MCAuto公司根据实体造型的数据制定一个文件规范，包括实体的CSG表示和边界表示，于1979年完成。
　　同时，美国国家标准局(NBS)和一些工业部门合作开发了初始图形交换规范(IGES)。IGES草案(IGES 1.0 )于1980年发表。IGES最初仅限于画蓝图所需要的典型几何图形和标注实体。1980年春季，ANSI接受IGES作为产品数据交换标准的一部分。
　　法国的航空航天业于1983年发表了在IGES基础上自行开发的数据交换规范SET(Standard d'Exchange et de Transfert)。成功地应用于"空中客车"计划和雷诺、标致等汽车公司。
　　德国的汽车制造业鉴于IGES的不足，制定了产品数据交换标准VDAFS(Verban der Deutschen Automobilindustrie-Flachennittstelle)，即DIN66301。1986年成为德国国家标准。
　　美国空军于1982-1983年组织实施了产品定义数据接口PDDI ( Product Definition Data Interface)计划。从产品的需求、设计、系统实现和PDDI原型系统的演示中确定了产品制造所需要的数据。PDDI第一次提到了产品数据和产品生命周期数据这些概念。
　　1984年，IGES组织设置了一个研究计划，称为PDES(Product Data Exchange Specification)。PDES计划着重于研究产品模型信息的交换。PDES提出三层体系结构，即应用层、逻辑层和物理层，制定并使用了形式化产品模型描述语言EXPRESS。无论是开发标准的方法论还是标准的结构和内容，PDES计划都有重大的突破和创新，为STEP标准的制订奠定了良好的基础。
　　1983年12月，国际标准化组织ISO设置了184委员会(TC184)。TC184名为工业自动化系统。TC184下设第四委员会(SC4)。SC4的领域是产品数据表达与交换。ISO TC184/SC4制订的标准称为STEP(STandard for the Exchange of Product model data)。1988年ISO把美国的PDES文本作为STEP标准的建议草案公布。PDES计划从PDES的制订转向STEP标准的应用，PDES也因此改名为"Product Data Exchange using STEP".
　　1990年6月，ISO TC184/SC4 制订了一份完成STEP文本的计划。计划包括TC184/SC4的组织结构、STEP文本的划分、STEP文本的确认过程等。
